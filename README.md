# Voice helper for YM

Android-приложение с **офлайн-распознаванием речи** на базе [Vosk](https://alphacephei.com/vosk/) (библиотека `vosk-android`). Есть режим распознавания в активности и **фоновый сервис** с постоянным прослушиванием и управлением громкостью при триггерных словах.

## Требования

- **Android Studio** Ladybug или новее (или JDK 11+ и Android SDK с API **36** для сборки).
- **Устройство или эмулятор** с Android **7.0** (API 24) и выше.
- Для работы распознавания нужна **полная модель Vosk** в `app/src/main/assets/model/` (см. ниже).

## Сборка и запуск

1. Клонируйте репозиторий и откройте папку проекта в Android Studio.
2. Дождитесь синхронизации Gradle (скачивание зависимостей).
3. Подключите устройство с отладкой по USB или выберите эмулятор.
4. Нажмите **Run** (▶) или выполните в корне проекта:

   ```bash
   ./gradlew assembleDebug
   ```

   Готовый APK: `app/build/outputs/apk/debug/app-debug.apk`.

5. При первом запуске выдайте разрешения **микрофон** и (на Android 13+) **уведомления** — они нужны для записи звука и для фонового сервиса.

## Использование

### Главный экран

- **Начать / Остановить распознавание** — распознавание в рамках открытого приложения; текст показывается на экране.
- **Запустить фоновый сервис** — поднимает `VoiceService` (foreground): постоянное прослушивание в фоне, уведомление в шторке. Остановка — кнопка **Остановить фоновый сервис** или через настройки системы.
- Переключатель **триггерное слово** — если включён, для команд в фоне по умолчанию нужны слова вроде «помощник», «ассистент» и т.п.; после срабатывания триггера в той же сессии команды можно говорить без него. Если выключить переключатель, голосовые команды обрабатываются **без** обязательного триггерного слова (см. `VoiceService` и `matchesCommand`).
- **Настройки микрофона** — экран с параметрами чувствительности, источника аудио, шумоподавления и др. (сохраняются в настройках приложения).

### Модель Vosk

При первом запуске приложение **копирует** содержимое `assets/model/` во внутреннее хранилище (`filesDir/model/`). В `assets/model/` должна лежать **распакованная** модель (папки `am`, `conf`, `graph`, `ivector` и необходимые файлы).

Если модели нет или она неполная, инициализация завершится ошибкой. Скачайте подходящую модель с [страницы моделей Vosk](https://alphacephei.com/vosk/models) (для русского языка — например `vosk-model-small-ru` или большую по необходимости), распакуйте и положите файлы в `app/src/main/assets/model/` с сохранением структуры каталогов.

## Структура проекта (кратко)

| Компонент | Назначение |
|-----------|------------|
| `MainActivity` | UI, кратковременное распознавание, запуск/остановка сервиса |
| `VoiceService` | Foreground-сервис, фоновое распознавание и обработка команд |
| `VoskRecognizer` | Работа с Vosk и микрофоном |
| `MicSettingsActivity` | Настройки микрофона |
| `ModelCopier` | Копирование модели из assets при первом запуске |
